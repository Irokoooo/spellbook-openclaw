## 1. SpellBook 是什么

SpellBook 是一个让普通人也能用上 AI Agent 的工作台。你可以：

- **像聊天一样派活**：在网页上发一条消息，AI 会自动联网、操作文件、爬取数据
- **把常用操作存成 Skill**：把反复做的任务固化成一键触发的"技能"，下次一句话就能调用
- **查看历史对话**：所有任务记录自动保存，随时回顾

整个系统分两部分：
- **网页端**：你日常打开的操作界面，部署在云端，随时随地访问
- **本地 Agent**：跑在你自己电脑上的程序，负责真正执行任务（访问本地文件、调用 AI）

---

## 2. 整体架构一图看懂

```
你的浏览器
    │  发消息 / 触发 Skill
    ▼
SpellBook 网页端（Vercel 云端）
    │  通过 Supabase 数据库派发任务
    ▼
本地 Agent（跑在你的 Windows 电脑上）
    │  调用 OpenClaw CLI
    ▼
DeepSeek AI（或其他兼容 OpenAI 的模型）
    │  返回结果
    ▼
结果写回 Supabase → 网页端实时显示
```

## 3. 第一步：部署网页端（管理员）

> 如果已有人帮你部署好网页端，跳过本节直接看第 4 步。

### 3.1 准备 Supabase 数据库

1. 注册 [supabase.com](https://supabase.com)，新建项目
2. 进入项目 → **SQL Editor**，粘贴并运行以下文件的内容：
   - `scripts/supabase_history_migration.sql`（创建对话历史表）
   - 其他建表 SQL（agents、skills、tasks 等，由部署者提供）
3. 记录以下三个值（在 **Project Settings → API** 里）：
   - `Project URL`
   - `anon public key`
   - `service_role secret key`（**不要泄露**，只用于服务端）

### 3.2 部署到 Vercel

1. Fork 本项目到自己的 GitHub
2. 在 [vercel.com](https://vercel.com) 导入该仓库
3. 在 Vercel 项目 **Settings → Environment Variables** 里填入：

   | 变量名 | 值 | 说明 |
   |---|---|---|
   | `NEXT_PUBLIC_SUPABASE_URL` | Supabase Project URL | 公开，前端可用 |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | anon public key | 公开，前端可用 |
   | `SUPABASE_SERVICE_ROLE_KEY` | service_role secret key | **仅服务端**，绝不暴露给浏览器 |
   | `NEXT_PUBLIC_APP_URL` | 你的 Vercel 域名，如 `https://spellbook.vercel.app` | |
   | `AI_API_KEY` | AI 提供商的 API Key | **仅服务端**，必填 |
   | `AI_BASE_URL` | API 基础地址（默认 `https://api.deepseek.com/v1`）| 可选，换模型时修改 |
   | `AI_MODEL` | 模型名称（默认 `deepseek-chat`）| 可选，如 `gpt-4o`、`qwen-max` |

4. 点击部署，等待完成

## 4. 第二步：配置 AI 接口

本地 Agent 需要一个兼容 OpenAI 格式的 AI 接口。默认使用 **DeepSeek**。

### 4.1 获取 DeepSeek API Key

1. 注册 [platform.deepseek.com](https://platform.deepseek.com)
2. 进入 **API Keys** 页面，创建一个新 Key
3. 复制保存（只显示一次）

### 4.2 写入 OpenClaw 配置

找到或创建文件 `C:\Users\你的用户名\.openclaw\.env`，填入：

```env
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

如果使用其他兼容 OpenAI 的模型（如 Qwen、Moonshot 等），还需要额外设置 Base URL：

```env
OPENAI_API_KEY=你的Key
OPENAI_BASE_URL=https://api.moonshot.cn/v1
```

> `.openclaw` 目录通常在 `C:\Users\你的用户名\.openclaw\`，如不存在请手动创建。

## 5. 第三步：安装本地 Agent

### 5.1 前置条件

确保电脑已安装：
- **Python 3.10 或更高版本**：[python.org](https://python.org)（安装时勾选"Add to PATH"）
- **Node.js 18 或更高版本**：[nodejs.org](https://nodejs.org)

### 5.2 在 SpellBook 网页端注册 Agent

1. 打开 SpellBook 网页，登录账号
2. 点击左下角 **「设置」**
3. 在「本地 Agent」区域，输入一个名称（如"办公室电脑"），点击 **「注册并下载配置」**
4. 浏览器会自动下载 `config.toml` 文件

### 5.3 安装并启动

1. 将下载的 `config.toml` 放到 `apps/local-agent/` 目录下
2. 双击运行 `install.bat`（首次安装，自动安装 Python 依赖和 openclaw CLI）
3. 安装完成后，双击 `start_agent.bat` 启动 Agent

终端显示类似以下内容说明启动成功：

```
[INFO] Starting SpellBook Agent: 办公室电脑 (xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx)
[INFO] Heartbeat OK — status: online
```

此时在 SpellBook 设置页，该 Agent 的状态会变为 **「在线」**（绿色）。

### 5.4 config.toml 说明

`config.toml` 由网页端自动生成，通常无需手动修改。完整字段说明：

```toml
# Supabase 连接信息（自动填入，勿改）
supabase_url = "https://xxx.supabase.co"
supabase_anon_key = "eyJhbGci..."
agent_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"   # 你的 Agent ID

# 心跳间隔（秒），Agent 多久向服务器汇报一次在线状态
heartbeat_interval = 30

# 轮询任务的间隔（秒），Agent 多久检查一次有没有新任务
poll_interval = 5

# openclaw 可执行文件路径
# 如果 install.bat 安装成功，保持默认 "openclaw" 即可
# 如果报错找不到命令，填写完整路径，如：
# openclaw_path = "C:/Users/你的用户名/AppData/Roaming/npm/openclaw.cmd"
openclaw_path = "openclaw"

# 本地 SQLite 数据库路径（存储任务执行记录）
local_db_path = "./data/db.sqlite"
```

## 6. 功能详解

### 6.1 AI 对话（聊天页）

打开 SpellBook 首页，直接在输入框发消息即可。

**工作流程：**

1. 发送消息后，AI 会先展示一个**执行计划**（3-5 条步骤）
2. 确认计划后点击 **「确认，开始执行」**，或者直接输入修改意见（如"改成输出 Excel 格式"）再确认
3. AI 开始执行，期间显示进度提示
4. 执行完毕后显示结果

**支持的任务类型（示例）：**
- "帮我爬取XX网站上的二手房数据，整理成表格"
- "搜索今天的科技新闻，总结成一段话"
- "读取桌面上的 data.xlsx，计算各列的平均值"

**深度思考模式：**
右下角可切换思考模式（关闭/高强度），高强度模式对复杂推理任务效果更好，但响应较慢。

### 6.2 历史对话

点击聊天页 header 右侧的**历史按钮**（时钟图标），从右侧滑出历史记录列表。

- 显示最近 50 条对话记录
- 点击任意一条，右侧聊天区会恢复该对话的完整内容
- 点击 **「新对话」** 开始一段全新对话

### 6.3 Skills 管理

Skills 是将重复任务固化为一键触发的"技能模板"。

**自动从对话中提炼 Skill：**

当 AI 完成一次实操任务后，会自动提议将该操作保存为 Skill：

```
📌 检测到可沉淀的 Skill
- 名称建议：二手房数据爬取
- 描述：爬取指定城市的链家二手房成交数据
- 参数：城市、时间范围、输出格式

帮你存为 Skill 方便下次一键触发吗？
```

回复"是"或"保存"即可存储。

**手动创建 Skill：**

1. 点击左侧导航 **「Skills」**
2. 点击右上角 **「新建 Skill」**
3. 填写名称、描述和输入参数，保存

**使用 Skill：**

在聊天框直接说"帮我执行XXX Skill"，AI 会自动识别并引导你填写所需参数后执行。

**启用/禁用 Skill：**

Skills 列表里每条右侧有开关，关闭后该 Skill 不会出现在 AI 的可用列表中。

### 6.4 Agent 管理（设置页）

- **注册新 Agent**：填写名称 → 「注册并下载配置」→ 把 `config.toml` 放到本地 Agent 目录
- **在线状态**：绿色"在线" = Agent 正在运行；灰色"离线" = Agent 未启动
- **重新下载配置**：点击 Agent 旁边的「config.toml」按钮，重新下载配置文件
- **删除 Agent**：删除后该 Agent 无法再接收任务，本地运行的进程会在下次心跳时自动停止

## 7. 常见问题

**Q：Agent 显示离线，但我已经启动了 `start_agent.bat`？**

检查 `config.toml` 里的 `supabase_url` 和 `supabase_anon_key` 是否正确。可以关掉再重启 `start_agent.bat`，观察终端输出有没有报错。

**Q：执行任务时报"AI 服务余额不足"？**

去 [platform.deepseek.com](https://platform.deepseek.com) 充值，或换一个有余额的 API Key，更新 `~/.openclaw/.env` 后重启 Agent。

**Q：执行任务时报"API Key 无效"？**

检查 `~/.openclaw/.env` 里的 `DEEPSEEK_API_KEY` 是否正确，注意不要有多余空格。

**Q：想换用其他 AI 模型怎么办？**

在 `~/.openclaw/.env` 里设置 `OPENAI_BASE_URL` 和对应的 `OPENAI_API_KEY`，重启 Agent 生效。

**Q：爬虫任务经常失败？**

Agent 内置了多层反爬绕过策略（换 UA、加延迟、找移动版、用搜索引擎替代等），如果依然失败，尝试换一种描述方式或拆分任务。

**Q：关闭电脑后 Agent 需要重新启动吗？**

是的，目前需要每次开机后手动双击 `start_agent.bat`。可以将其设置为开机自启（Win + R → `shell:startup` → 创建快捷方式）。

**Q：SQL 迁移报错 "policy already exists"？**

重新运行最新的 `scripts/supabase_history_migration.sql`，该文件已做幂等处理，可以重复执行。
