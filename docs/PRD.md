# SpellBook 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 背景
客户是非技术背景的业务人员，希望使用 OpenClaw 提升工作效率，但面临：
- OpenClaw 配置复杂（XML/JSON/Markdown）
- 需要命令行操作，学习成本高
- 业务场景需要外挂脚本（爬虫、企微API）

### 1.2 目标用户
| 角色 | 需求 |
|------|------|
| 业务用户（客户） | 零门槛使用 AI 技能，手机/电脑随时调用 |
| 技术运营（你） | 快速定制 Skill，管理多个客户的 Agent |

### 1.3 核心价值主张
**"手机下单，电脑干活，数据不出本地"**

---

## 2. 业务场景与优先级

| P | 场景 | 描述 | MVP 范围 |
|---|------|------|---------|
| P0 | 基础对话 | Web 界面调用 OpenClaw 对话 | ✅ |
| P0 | 任务队列 | 离线下发任务，Agent 上线后执行 | ✅ |
| P1 | 市场盘点 | 爬虫 + 结构化提取 + 分析 | ✅ 框架 |
| P1 | 解决方案 | RAG 检索 + 文档生成 | ✅ 框架 |
| P2 | 客户运营 | 企微 Webhook + 标签管理 | ❌ 待定 |
| P3 | 投资策略 | 行情监控 + 策略提醒 | ❌ 不做 |

## 3. 功能需求

### 3.1 Web 控制台

#### 3.1.1 用户认证
- [ ] 邮箱/密码登录（Supabase Auth）
- [ ] 多租户：每个用户独立的 Agent 和 Skill 空间

#### 3.1.2 聊天界面
- [ ] 选择已配置的 Skill 发起对话
- [ ] Skill 对应的动态表单（如：输入公司名称）
- [ ] 流式输出 AI 回复
- [ ] 对话历史记录

#### 3.1.3 任务中心
- [ ] 查看待执行/执行中/已完成任务
- [ ] 任务详情：输入、输出、执行日志
- [ ] 取消排队中的任务

#### 3.1.4 Skill 管理
- [ ] Skill 列表（启用/禁用）
- [ ] Skill 编辑器（Markdown + 表单配置）
- [ ] Skill 测试沙盒

#### 3.1.5 知识库管理（P1）
- [ ] 上传文档（PDF/Word/PPT）
- [ ] 查看文档列表和索引状态
- [ ] 关联 Skill 使用特定知识库

### 3.2 Local Agent

#### 3.2.1 核心功能
- [ ] 与云端 WebSocket 保持连接
- [ ] 拉取并执行任务队列
- [ ] 调用本地 OpenClaw 执行 Skill
- [ ] 回传执行结果

#### 3.2.2 扩展能力
- [ ] 执行 Python 脚本（爬虫等）
- [ ] 本地 RAG 检索
- [ ] （未来）企微 Webhook 监听

---

## 4. 非功能需求

### 4.1 安全性
- 敏感数据（文档、对话内容）仅存本地
- 云端只存任务元信息和状态
- Agent 与云端通信加密（WSS）

### 4.2 可用性
- Agent 离线时任务自动排队
- Agent 上线后自动恢复执行
- Web 端显示 Agent 在线状态

### 4.3 可扩展性
- 支持未来接入多个 Agent（多台电脑）
- Skill 格式兼容 OpenClaw 原生格式

## 7. 技术债务和风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 企微 API 权限 | 场景3无法实现 | 先做其他场景，待客户确认权限 |
| 反爬难度未知 | 场景1效果不稳定 | 龙虾动态调试 + 人工兜底 |
| Agent 离线过久 | 任务积压 | 设置任务过期时间，通知用户 |
